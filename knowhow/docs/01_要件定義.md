# ノウハウ共有AI音声アシスタント - 要件定義書

**プロジェクト名**: AIでノウハウを組織の資産へ
**クライアント**: JFEシビル株式会社
**プラットフォーム**: Web/PWA（Azure完結）
**作成日**: 2026-02-11

---

## 1. プロジェクト概要

建設・土木現場における熟練者のノウハウ（暗黙知）をAI技術で組織全体の資産として共有・活用するソリューション。AI音声アシスタントを中核技術とし、音声によるナレッジの蓄積・検索・提供を実現する。

### 1.1 解決すべき課題

| 課題 | 現状 | 目標 |
|------|------|------|
| 現場間のノウハウ孤立 | 各現場（A〜D）が独立運営、知識が共有されない | 全現場のナレッジを一元化 |
| 熟練者と新人の格差 | 新人はマニュアル依存、応用力不足 | AIを通じて熟練者の判断力・経験を活用 |
| 熟練者のチェック漏れ | 慣れによる見落とし、類災防止への感受性低下 | AIによる自動チェック・アラート |
| トラブル増加・費用増大 | ノウハウ非共有による品質バラつき | 予防的な知識活用でトラブル・コスト削減 |

### 1.2 熟練者 vs 新人の決定的な差

| 項目 | 熟練者 | 新人 |
|------|--------|------|
| 判断力 | 意図を持った判断 | なんとなく使いまわし |
| 最適化 | 経験に基づく最良判断 | 背景・理由が不明 |
| トラブル対応 | 予測能力あり | 後手対応 |
| コスト管理 | コスト意識が高い | 無駄なコスト発生 |
| 柔軟性 | 応用力・柔軟性 | マニュアル依存 |

## 3. 機能要件

### 3.1 AI音声ソリューション（コアフロー）

```
STEP 1: ノウハウ（ルール）収集
  └─ 現場データ・過去データから熟練者のノウハウを収集

STEP 2: AI解析・データベース化
  └─ AI解析 → ベクトル化 → ナレッジDB格納

STEP 3: 音声ガイドで知識共有
  └─ 音声アシスタントによるプッシュ型提供

STEP 4: 現場で実践
  └─ 実践結果をフィードバック → 継続的改善
```

### 3.2 利用モード

#### モード1: 事務作業モード（施工検討・事前準備）

| 機能 | 説明 |
|------|------|
| AIチャットボット質問 | チェック項目を対話形式で一つずつ確認 |
| 類災事例の自動提示 | 過去の類似事故・トラブル事例をAIが自動検索して表示 |
| リスク評価アシスト | 計画内容に基づくリスク分析と注意点の提示 |
| 施工計画レビュー | AIによる計画書のチェックポイント確認 |

#### モード2: 現場作業モード（補修・点検作業中）

| 機能 | 説明 |
|------|------|
| 音声状況報告 | 作業開始時に音声で現場状況を報告 |
| リアルタイム音声アドバイス | 作業中にAIが音声でアドバイス提供 |
| 類似事例の音声提示 | 過去の類似事例を音声で案内 |
| ハンズフリー操作 | 現場で手が使えない状況でも音声で操作可能 |

### 3.3 ナレッジ管理機能

| 機能 | 説明 |
|------|------|
| ノウハウ登録 | 熟練者が音声/テキストでノウハウを登録 |
| カテゴリ分類 | 工種・作業種別・リスク種別による自動分類 |
| タグ付け | AIによる自動タグ付け + 手動タグ編集 |
| バージョン管理 | ナレッジの更新履歴管理 |
| 承認ワークフロー | 登録されたノウハウの品質チェック・承認フロー |

### 3.4 検索・推薦機能

| 機能 | 説明 |
|------|------|
| セマンティック検索 | ベクトル検索による意味ベースのナレッジ検索 |
| 類似事例検索 | 現在の作業に類似した過去事例の自動検索 |
| プッシュ型通知 | AIが能動的に関連ナレッジを提案（理想形） |
| コンテキスト連動 | 現場・工種・作業内容に応じた自動絞り込み |

### 3.5 分析・レポート機能

| 機能 | 説明 |
|------|------|
| ナレッジ活用状況 | どのノウハウがどれだけ活用されているか |
| ユーザー別学習進捗 | 新人の成長度合いの可視化 |
| リスク傾向分析 | 現場別・工種別のリスク傾向 |
| ナレッジギャップ分析 | 未カバーの領域の特定 |

## 6. データモデル（概要）

### 6.1 主要エンティティ

```
ユーザー (users)
├── id, email, password, name, role
├── department, site_id
└── manager_id（階層管理）

ナレッジ (knowledge_items)
├── id, title, content, category
├── author_id, status (draft/review/published)
├── work_type（工種）, risk_level
├── embedding（ベクトル）
└── created_at, updated_at

類災事例 (incident_cases)
├── id, title, description, cause, countermeasure
├── site, work_type, severity
├── embedding
└── occurred_at

チェックリスト (checklists)
├── id, name, work_type
└── items[] (check_item, priority, related_knowledge_id)

利用ログ (usage_logs)
├── id, user_id, knowledge_id, action_type
├── context (site, work_type)
└── timestamp

音声セッション (voice_sessions)
├── id, user_id, mode (office/field)
├── transcript, ai_response
├── related_knowledge_ids[]
└── created_at
```

## 7. UI/UX要件

### 7.1 デザイン方針

- **モバイルファースト**: 現場でのスマートフォン利用を最優先
- **音声ファースト**: 現場作業中は手が使えないため音声I/Oが基本
- **大きなボタン**: 手袋着用時でも操作可能なUI
- **高コントラスト**: 屋外の直射日光下でも視認可能
- **2モード切替**: 事務作業モード ⇔ 現場作業モードのワンタップ切替

### 7.2 画面構成

1. **ログイン画面** - メール/パスワード認証
2. **ホーム画面** - モード選択（事務/現場）、最近のナレッジ、通知
3. **事務作業モード画面** - チャットボット形式のチェック確認UI
4. **現場作業モード画面** - 音声中心の大きなマイクボタン、音声波形表示
5. **ナレッジ登録画面** - 音声/テキストでのノウハウ入力
6. **ナレッジ検索画面** - キーワード/音声検索
7. **ナレッジ詳細画面** - 内容表示、関連事例、音声読み上げ
8. **分析ダッシュボード** - 活用状況、リスク傾向

## 9. sales-appからの技術流用可能性

### 9.1 そのまま流用可能

| 技術 | sales-appでの実績 | knowhowでの用途 |
|------|------------------|----------------|
| Azure OpenAI (GPT-4) | AIヒアリング・動的フォローアップ | AIチャットボット・ナレッジ解析 |
| Azure Speech SDK | 音声入力（ja-JP STT/TTS） | 音声アシスタント（STT/TTS） |
| React + Vite + PWA | モバイルファーストUI、オフライン対応 | 同様のPWA構成 |
| Node.js + Express | APIサーバー | 同様のAPIサーバー |
| PostgreSQL | データ永続化 | ナレッジDB |
| Redis | キャッシュ（フォールバック対応済み） | セッション・検索キャッシュ |
| JWT認証 | 営業/マネージャー権限管理 | ユーザー種別権限管理 |
| Socket.IO | リアルタイム音声通信 | リアルタイム音声アシスタント |
| Docker + Azure App Service | コンテナデプロイ | 同様のデプロイ構成 |
| GitHub Actions CI/CD | 自動デプロイ | 同様のCI/CDパイプライン |
| Workbox PWA | オフライン対応 | 現場でのオフライン対応 |

### 9.2 拡張・追加が必要

| 技術 | 理由 |
|------|------|
| Azure Cognitive Search（ベクトル検索） | セマンティックなナレッジ検索に必要 |
| Azure OpenAI Embeddings | ナレッジのベクトル化に必要 |
| プッシュ通知（Azure Notification Hubs） | プッシュ型のナレッジ提案に必要 |
| 承認ワークフロー機能 | ナレッジの品質管理に必要 |
| 騒音環境対応の音声処理 | 建設現場特有の要件 |

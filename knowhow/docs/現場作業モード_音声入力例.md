# 現場作業モード - 音声入力例集

**対象**: 現場作業者（新人〜熟練者）が実際に音声で話しかける内容の想定例
**実装状態**: FieldVoicePage.jsx で全対応済み（Web Speech API + Azure TTS）
**最終更新**: 2026-02-12

---

## 7. 想定される会話の流れ（対話例）

### 例1: 新人がコンクリート打設前に確認

> **作業者**: 「今日初めてコンクリートの打設をやるんですけど、確認しておくことを教えてください」
>
> **AI**: 「コンクリート打設前の確認事項をお伝えします。まず安全面から。作業範囲の立入禁止区画の設置、ポンプ車の設置位置と旋回範囲の確認をしてください。次に品質面。型枠の寸法と固定状態、鉄筋のかぶり厚さ、スペーサーの配置を確認してください。過去に型枠のセパレーター不足でコンクリートが漏れた事例がありますので、特にご注意ください。」
>
> **作業者**: 「スランプの受入検査はどうやればいいですか」
>
> **AI**: 「スランプ試験はアジテータ車から試料を採取し、スランプコーンに3層詰めで各25回突きます。許容値は指定スランプ±2.5センチです。今日の気温が高い場合はスランプロスに注意してください。」

### 例2: トラブル発生時

> **作業者**: 「掘削中に想定外の地下水が出てきました。どうすればいいですか」
>
> **AI**: 「まず作業を一旦停止してください。安全確認が最優先です。次に以下を確認してください。水量はどの程度ですか？湧き出しか噴出しかで対応が変わります。過去に同様のケースでは、釜場排水で対応した事例があります。水量が多い場合はウェルポイント工法の検討が必要です。監督者への報告はお済みですか？」
>
> **作業者**: 「じわじわ湧き出してる感じです。量はそこまで多くないです」
>
> **AI**: 「了解しました。湧水量が少なければ釜場排水で対応できます。掘削底面の隅に50センチ角程度の集水桝を設けて、水中ポンプで排水してください。ただし、排水先の確認と濁水処理が必要です。また地盤の緩みに注意して、法面の状態を定期的に確認してください。」

### 例3: ベテランのチェック漏れ防止

> **作業者**: 「明日から足場の解体に入ります。いつもの手順で大丈夫だと思うけど、念のため確認」
>
> **AI**: 「足場解体の確認事項です。見落としやすいポイントを中心にお伝えします。解体順序は上層から順に、壁つなぎは解体する層のみ撤去してください。過去に壁つなぎを先に全撤去して足場が倒壊した事例があります。また、解体部材の落下防止措置と、第三者への飛来落下防止ネットの確認もお願いします。明日の天候予報は確認されていますか？」

## 補足: 音声入力の特性を考慮した設計ポイント

| 特性 | 対応方針 | 実装状態 |
|------|---------|---------|
| 騒音環境（重機・工具の音） | 短い文での入力を推奨、聞き取れなかった場合は確認 | 実装済み（テキスト入力フォールバック） |
| 専門用語の認識ミス | AI側で建設用語の補正を実施 | 実装済み（/api/ai/correct-speech） |
| 方言・略語 | 「鉄筋→てっきん」「型枠→かたわく」等の揺れに対応 | Azure OpenAI で対応 |
| 緊急時の短い発話 | 「水が出た！」「止まった！」等にも文脈から適切に対応 | 実装済み（コンテキスト連動） |
| 手が塞がっている状態 | 追加質問は最小限、一度の応答で要点を網羅 | 実装済み（簡潔応答プロンプト） |
| TTS再生中のエコー | VADエコーキャンセルで自分の音声再生を無視 | 実装済み |

## 技術仕様

| 項目 | 仕様 |
|------|------|
| 音声認識エンジン | Web Speech API（ja-JP） |
| 音声合成エンジン | Azure Speech TTS（Nanami Neural Voice） |
| 音声再生方式 | Web Audio API（AudioContext.decodeAudioData） |
| フォールバックTTS | ブラウザ内蔵 SpeechSynthesis API |
| 音声入力タイムアウト | 30秒 |
| 対応ブラウザ | Chrome, Edge, Safari（iOS含む） |
